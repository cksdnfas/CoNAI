# 감시 폴더와 백업 소스

감시 폴더는 CoNAI가 기존 파일 위치를 스캔해 라이브러리에 등록하는 기능입니다. 백업 소스는 외부 원본 위치에서 CoNAI Upload 내부 대상으로 복사하거나 WebP로 변환해 가져오는 기능입니다.

## 관련 페이지

| 목적 | 경로 |
| --- | --- |
| 감시 폴더 설정 | `/settings` → 감시 폴더 |
| 감시폴더 그룹 보기 | `/groups?tab=folders` |
| 홈 피드 | `/` |
| 데이터 경로/백업 | [데이터 경로와 백업](./DATA_PATHS_AND_BACKUP.md) |

## 감시 폴더와 백업 소스 차이

| 항목 | 감시 폴더 | 백업 소스 |
| --- | --- | --- |
| 목적 | 기존 폴더를 스캔해 등록 | 외부 원본을 Upload 내부로 가져오기 |
| 파일 위치 | 원본 위치를 참조 | 대상 폴더로 복사/변환 |
| 그룹 | 감시폴더 그룹으로 자동 구조화 | 가져온 결과는 일반 등록 파일처럼 취급 |
| 주요 작업 | 스캔, 전체 재스캔, watcher | copy original, convert WebP, watcher |

## 감시 폴더 추가

1. `/settings` → 감시 폴더를 엽니다.
2. **감시 폴더 추가**를 누릅니다.
3. 폴더 경로를 입력합니다.
4. 표시 이름을 입력합니다.
5. 필요한 옵션을 설정합니다.
6. **경로 검증**으로 CoNAI가 읽을 수 있는지 확인합니다.
7. **감시 폴더 추가**를 누릅니다.
8. 전체 스캔 또는 폴더 스캔을 실행합니다.

## 감시 폴더 옵션

| 옵션 | 설명 |
| --- | --- |
| 폴더 경로 | 실제 이미지/비디오가 있는 경로입니다. 예: `D:\Images\Incoming` |
| 표시 이름 | UI에 보일 이름입니다. 비워도 경로 기준으로 구분됩니다. |
| 스캔 주기(분) | 자동 스캔 간격입니다. 너무 짧으면 디스크 부하가 큽니다. |
| watcher polling(ms) | watcher 감시 간격입니다. 네트워크 드라이브는 길게 잡는 편이 안전합니다. |
| 제외 확장자 | `tmp, db`처럼 쉼표로 구분합니다. |
| 제외 패턴 | `@eaDir, cache`처럼 경로/이름 패턴을 제외합니다. |
| 자동 스캔 | 주기 기반 스캔을 켭니다. |
| 하위 폴더 포함 | 하위 폴더까지 재귀 스캔합니다. |
| watcher 시작/사용 | 변경 감시를 시작합니다. |
| 폴더 활성화 | 스캔 대상으로 사용할지 결정합니다. |

## 운영 카드 읽기

감시 폴더 운영 카드에는 다음 값이 표시됩니다.

- 등록된 폴더 수
- 등록된 백업 소스 수
- 감시 중 watcher 수
- watcher 오류 수
- 24시간 이벤트 수
- 최근 스캔 로그 대상

상단 작업:

| 버튼 | 기능 |
| --- | --- |
| 새로고침 | 감시 폴더, 백업 소스, 로그, watcher 상태를 다시 불러옵니다. |
| 전체 파일 검증 | DB 레코드와 실제 파일의 불일치를 확인합니다. |
| 전체 스캔 | 등록된 감시 폴더를 모두 스캔합니다. |

## 개별 폴더 작업

폴더 상세 모달에서 실행합니다.

| 작업 | 설명 |
| --- | --- |
| 폴더 스캔 | 해당 폴더만 스캔합니다. |
| watcher 시작 | 파일 변경 감시를 시작합니다. |
| watcher 중지 | 변경 감시를 멈춥니다. |
| watcher 재시작 | 오류나 경로 변경 후 watcher를 다시 시작합니다. |
| 폴더 설정 저장 | 이름, 주기, 제외 규칙, 활성화 상태를 저장합니다. |
| 폴더 제거 | 등록만 제거합니다. 원본 파일 삭제가 아닙니다. 기본 폴더는 제거가 제한됩니다. |

## 백업 소스 추가

1. `/settings` → 감시 폴더에서 **백업 소스 추가**를 누릅니다.
2. source 폴더 경로를 입력합니다.
3. 표시 이름을 입력합니다.
4. Upload 내부 대상 경로를 지정합니다.
5. 가져오기 모드를 선택합니다.
6. 필요하면 watcher와 하위 폴더 포함을 켭니다.
7. source 경로 검증 후 추가합니다.

## 백업 소스 옵션

| 옵션 | 설명 |
| --- | --- |
| source 폴더 경로 | 원본 파일이 있는 외부 위치입니다. |
| Upload 내부 대상 경로 | `Upload/` 아래 상대 경로입니다. 예: `Backup/001` |
| 가져오기 모드 | `원본 복사` 또는 `WebP 변환(메타 보존)` |
| WebP 품질 | WebP 변환 모드에서만 사용합니다. |
| 하위 폴더 포함 | source 하위 폴더까지 포함합니다. |
| watcher 시작 | source 변경 감시를 시작합니다. |

## 최근 스캔 로그

최근 스캔 로그는 폴더별 처리 결과를 보여줍니다.

| 값 | 의미 |
| --- | --- |
| 상태 | 스캔 성공/실패 상태 |
| 스캔 | 확인한 파일 수 |
| 신규 | 새로 등록된 파일 수 |
| 기존 | 이미 등록되어 있던 파일 수 |
| 오류 | 처리 실패 수 |
| 시각 | 스캔 실행 시각 |

## 감시폴더 그룹과 연결

감시 폴더 스캔 결과는 `/groups?tab=folders`에서 폴더 구조 기반 그룹으로 볼 수 있습니다.

폴더 구조가 바뀌었으면 그룹 페이지의 감시폴더 그룹 루트에서 재구성을 실행하세요.

## 운영 팁

- 대용량 폴더는 처음에 watcher보다 전체 스캔으로 안정화합니다.
- 네트워크 드라이브는 polling 간격을 길게 두고 오류 로그를 확인합니다.
- 제외 패턴을 너무 넓게 잡으면 이미지가 홈에 안 보입니다.
- 폴더 제거는 원본 파일 삭제가 아닙니다. 실제 파일 정리는 별도로 관리합니다.
- DB/파일 불일치가 의심되면 전체 파일 검증을 먼저 실행합니다.
