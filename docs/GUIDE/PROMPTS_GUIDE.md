# 프롬프트 관리

프롬프트 페이지는 이미지에서 추출된 긍정/부정/자동 프롬프트를 검색하고, 그룹화하고, 재사용하는 화면입니다.

## 관련 페이지

| 목적 | 경로 |
| --- | --- |
| 프롬프트 관리 | `/prompts` |
| 이미지 상세 | `/images/:compositeHash` |
| 이미지 생성 | `/generation` |
| 와일드카드 생성 | `/prompts` → 와일드카드 탭 |

## 상단 탭

| 탭 | 설명 |
| --- | --- |
| 긍정 | 이미지에서 추출된 positive prompt 항목입니다. |
| 부정 | negative prompt 항목입니다. |
| 자동 | 자동 태그/분류 결과입니다. 보호된 자동 그룹 항목은 직접 변경이 제한됩니다. |
| Danbooru | Danbooru 기준 탐색/그룹화용 화면입니다. |
| 와일드카드 | 권한이 있을 때 표시되는 와일드카드 생성 패널입니다. |
| 프리셋 | 프롬프트 프리셋 관리 화면입니다. |

## 기본 화면

프롬프트 목록 화면은 왼쪽 그룹 트리와 오른쪽 목록으로 구성됩니다.

- 그룹 트리: 프롬프트 그룹 계층과 항목 수
- 목록: 프롬프트 텍스트, 타입, 사용량, 소속 그룹
- 검색: 현재 타입/그룹 안에서 텍스트 검색
- 정렬: 사용량 등 기준과 오름/내림차순
- 페이지네이션: 많은 항목을 나눠 탐색

목록에서 프롬프트를 클릭하면 해당 텍스트가 클립보드에 복사됩니다.

## 그룹 작업

왼쪽 패널의 아이콘 버튼으로 다음 작업을 실행합니다.

| 작업 | 설명 |
| --- | --- |
| 상태/요약 | 프롬프트 통계와 상위 프롬프트를 확인합니다. |
| 수동 수집 | positive/negative prompt 텍스트를 직접 넣어 수집합니다. 자동 탭에서는 제한됩니다. |
| 그룹 추가 | 현재 타입의 프롬프트 그룹을 만듭니다. |
| Danbooru 자동 그룹 | Danbooru 기준으로 그룹을 구성합니다. |
| 내보내기 | 현재 타입의 그룹 구조를 JSON으로 내보냅니다. |
| 가져오기 | JSON 그룹 구조를 가져옵니다. |
| 위/아래 이동 | 같은 부모 아래에서 그룹 순서를 조정합니다. |
| 편집/삭제 | 선택한 그룹을 수정하거나 삭제합니다. |

그룹 삭제 시 하위 그룹도 함께 정리되고, 포함된 프롬프트는 Unclassified 계열로 이동할 수 있습니다.

## 프롬프트 수동 수집

1. 긍정 또는 부정 탭을 엽니다.
2. 왼쪽 패널에서 수동 수집 버튼을 누릅니다.
3. Positive prompt와 Negative prompt 중 하나 이상을 입력합니다.
4. 실행합니다.
5. 수집된 프롬프트가 목록과 그룹 통계에 반영되는지 확인합니다.

## 프롬프트 선택 작업

목록에서 여러 프롬프트를 선택하면 하단 선택 바가 열립니다.

| 작업 | 설명 |
| --- | --- |
| 그룹 지정 | 선택 프롬프트를 다른 그룹에 배정합니다. |
| 삭제 | 사용량이 남아 있지 않고 보호되지 않은 항목만 삭제합니다. |
| 선택 해제 | 현재 선택을 비웁니다. |

보호된 자동 그룹, Danbooru 기반 그룹, LoRA 보호 그룹 항목은 직접 수정/삭제가 제한될 수 있습니다.

## 검색과 정렬 팁

- 모델명/태그 일부만 떠오르면 검색창에 짧게 입력합니다.
- 자주 쓰는 프롬프트를 찾을 때는 사용량 기준 정렬을 사용합니다.
- 특정 그룹 안에서 검색하면 노이즈가 줄어듭니다.
- 목록에서 프롬프트를 클릭하면 바로 복사되므로 생성 페이지에 붙여넣기 쉽습니다.

## 이미지 상세와의 관계

이미지 상세의 프롬프트 영역은 프롬프트 그룹 데이터를 활용합니다.

- 상세에서 Plain/Group 표시 전환 가능
- Group 표시는 프롬프트 그룹 계층을 기준으로 태그를 묶음
- 자동 프롬프트는 자동 탭/자동 태그 추출 결과와 연결됨

## 생성 기능과의 관계

정리한 프롬프트는 생성 기능에서 재사용하기 좋습니다.

- NAI/Codex/ComfyUI 프롬프트 작성 시 복사해서 사용
- 와일드카드 생성으로 변형 후보 만들기
- 프리셋으로 반복 입력 줄이기
