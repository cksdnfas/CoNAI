# 처음 시작하기

CoNAI는 생성 이미지와 비디오를 로컬에서 모으고, 검색하고, 분류하고, 다시 생성 흐름으로 되돌리는 작업대입니다.

## 가장 짧은 사용 흐름

1. 설치합니다.
2. `.env`를 준비합니다.
3. 앱을 실행합니다.
4. 관리자 계정 또는 게스트 계정을 설정합니다.
5. 감시 폴더를 등록합니다.
6. 전체 스캔을 실행합니다.
7. 홈에서 이미지가 보이는지 확인합니다.
8. 이미지 상세에서 메타데이터와 유사도를 확인합니다.
9. 그룹/프롬프트/생성 기능으로 확장합니다.

## 처음 볼 페이지

| 목적 | 페이지 |
| --- | --- |
| 접근 가능한 화면 확인 | `/access` |
| 이미지 피드 보기 | `/` |
| 파일 직접 등록/메타 추출 | `/upload` |
| 폴더 자동 등록 | `/settings` → 감시 폴더 |
| 이미지 상세 | `/images/:compositeHash` |
| 이미지 생성 | `/generation` |
| 프롬프트 정리 | `/prompts` |
| 이미지 그룹 정리 | `/groups` |
| 앱 설정 | `/settings` |

## 핵심 개념

### 홈

등록된 이미지와 비디오를 보는 기본 피드입니다. 선택, 다운로드, 그룹 추가, 삭제가 여기서 시작됩니다.

### 감시 폴더

이미지가 이미 쌓이는 폴더를 CoNAI가 스캔하도록 등록하는 기능입니다. ComfyUI/NovelAI 결과 폴더, 수동 보관 폴더, 네트워크 드라이브를 연결할 때 사용합니다.

### 업로드

파일을 직접 끌어다 놓아 등록하거나, 단일 이미지의 메타데이터만 추출해 확인하는 페이지입니다.

### 이미지 상세

이미지 한 장의 원본 경로, 모델, 프롬프트, 자동 태그, 작가 후보, 중복/유사 이미지를 확인합니다.

### 그룹

이미지를 폴더 구조 또는 사용자 정의 기준으로 묶습니다. 감시폴더 그룹은 자동 구조 확인용, 커스텀 그룹은 작업/선별용으로 쓰는 것이 좋습니다.

### 프롬프트

이미지에서 추출된 긍정/부정/자동 프롬프트를 검색하고 그룹화합니다. 상세 화면의 그룹 표시와 생성 작업 재사용에 연결됩니다.

### 생성

NAI, Codex, ComfyUI, 워크플로우 기반 생성을 실행하고 결과 이력을 확인합니다.

## 계정과 권한

CoNAI는 설정에 따라 로그인 없이 공개 화면만 보이거나, 관리자/게스트/일반 계정으로 접근 권한을 나눌 수 있습니다.

| 목적 | 위치 |
| --- | --- |
| 로그인 | `/login` |
| 접근 가능한 페이지 목록 | `/access` |
| 계정/권한 관리 | `/settings` → 보안 |
| 첫 계정 생성 | `/settings` → 보안 또는 로그인 초기 설정 흐름 |

외부 네트워크에 열기 전에는 관리자 계정, 게스트 권한, 삭제 권한을 먼저 정리하세요.
