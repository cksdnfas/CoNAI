# 이미지 메타데이터 편집

이미지 메타데이터 편집은 이미 등록된 정적 이미지의 프롬프트와 생성 정보를 수정하거나, 수정된 메타데이터가 들어간 새 파일을 다운로드하는 기능입니다.

## 관련 페이지

| 목적 | 경로 |
| --- | --- |
| 메타데이터 편집 | `/images/:compositeHash/metadata` |
| 이미지 상세 | `/images/:compositeHash` |
| 단일 파일 메타데이터 수정 다운로드 | `/upload` → 미리보기/추출 |

## 지원 범위

현재 편집 저장은 정적 이미지 파일 중심입니다.

- 지원: PNG, JPEG, WebP 같은 정적 이미지
- 제한: 비디오, 일부 애니메이션 파일, 원본이 누락된 항목

상세 화면에서 **메타데이터 편집** 버튼이 보이지 않으면 해당 항목이 편집 대상이 아니거나 권한/파일 타입 조건을 만족하지 않은 것입니다.

## 화면 구성

편집 화면은 두 영역으로 나뉩니다.

| 영역 | 설명 |
| --- | --- |
| 왼쪽 미리보기 | 대상 이미지, 복합 해시, 파일명을 확인합니다. |
| 오른쪽 편집 폼 | 출력 형식과 생성 메타데이터 값을 수정합니다. |

편집 폼 항목:

- 출력 형식: PNG, JPEG, WebP
- 프롬프트
- 네거티브 프롬프트
- 모델
- 스텝
- 샘플러

## 다운로드와 저장 차이

| 작업 | 결과 |
| --- | --- |
| 다운로드 | 원본 DB 항목은 유지하고, 수정된 메타데이터가 들어간 새 파일을 내려받습니다. |
| 저장 | 현재 등록된 이미지의 메타데이터를 저장합니다. 확인 창이 표시됩니다. |

안전하게 확인하려면 먼저 **다운로드**로 결과 파일을 받아보고, 내용이 맞으면 **저장**을 사용하세요.

## 저장 절차

1. 상세 화면에서 메타데이터 편집을 엽니다.
2. 프롬프트, 네거티브 프롬프트, 모델, 스텝, 샘플러를 수정합니다.
3. 출력 형식을 확인합니다.
4. 필요하면 **다운로드**로 수정 파일을 먼저 확인합니다.
5. **저장**을 누릅니다.
6. 확인 창에서 저장을 승인합니다.
7. 상세 화면으로 돌아가 새로고침해 값이 반영됐는지 확인합니다.

## 빈 값 처리

저장 시 비워 둔 필드는 삭제/초기화 대상으로 처리될 수 있습니다. 실수로 프롬프트나 모델명을 지우지 않도록 저장 전 폼을 다시 확인하세요.

## 업로드 페이지의 메타데이터 편집과 차이

| 위치 | 용도 |
| --- | --- |
| `/upload` 미리보기/추출 | 등록 전 단일 파일을 확인하고 수정 파일을 다운로드합니다. DB를 바꾸지 않습니다. |
| `/images/:hash/metadata` | 이미 등록된 이미지 기준으로 다운로드 또는 저장을 실행합니다. |

## 실패할 때 확인할 것

| 증상 | 확인할 것 |
| --- | --- |
| 편집 페이지가 열리지 않음 | composite hash가 올바른지, 상세 접근 권한이 있는지 확인 |
| 저장 버튼이 비활성 | 파일 타입이 정적 이미지인지 확인 |
| 저장 실패 | 원본 파일 경로, 파일 잠금, 쓰기 권한, RecycleBin/백업 정책 확인 |
| 다운로드 실패 | 브라우저 다운로드 차단, 출력 형식, 원본 파일 손상 여부 확인 |
| 값이 반영되지 않음 | 상세 화면 새로고침, 캐시, 재추출 큐 상태 확인 |
