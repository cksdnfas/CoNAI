# 설정 전체 지도

설정 페이지는 CoNAI의 런타임 동작, 폴더, 보안, 자동 추출, 저장 정책을 관리하는 관리자 중심 화면입니다.

## 관련 페이지

| 목적 | 경로 |
| --- | --- |
| 설정 | `/settings` |
| 접근 가능 페이지 확인 | `/access` |
| 로그인 | `/login` |
| 감시폴더 그룹 | `/groups?tab=folders` |

설정 페이지는 관리자이거나, 아직 인증 정보가 없는 초기 상태에서 열 수 있습니다. 일반 계정은 권한에 따라 접근이 제한됩니다.

## 설정 탭 요약

| 탭 | 주요 기능 | 먼저 볼 때 |
| --- | --- | --- |
| 일반 | 언어, RecycleBin, 삭제 보호, 유사/중복 검사, 데이터 재매칭 | 첫 실행/삭제 정책/유사도 기본값 |
| 감시 폴더 | 감시 폴더, 백업 소스, watcher, 전체 스캔, 파일 검증 | 이미지가 홈에 안 보일 때 |
| 외형 | 테마/표시 밀도/상세 관련 표시 옵션 | UI가 너무 크거나 작을 때 |
| 보안 | 계정, 권한 그룹, 페이지 권한, 복구 정보 | 외부 접속/다중 사용자 전 |
| 자동 | Kaloscope, WD Tagger, 평가 가중치/등급, 자동 테스트 | 자동 태그/작가 추출 실패 시 |
| 메타데이터 | PNG secondary extraction, stealth scan, 전체 재추출 | 프롬프트 추출이 누락될 때 |
| 이미지 저장 | 저장 포맷/품질/리사이즈, 생성 텀, 비디오 최적화 | 업로드/생성 결과 저장 방식 조정 |
| LLM 연결 | LLM/외부 연결 설정 | LLM 기능이나 연결형 기능 사용 전 |

## 일반 탭

주요 항목:

- 언어
- RecycleBin 경로
- 유사/중복 검사 정책
- 삭제 시 RecycleBin 보호
- 갤러리 기능
- 등급 배지
- 종료 시 캔버스 임시 데이터 자동 정리
- 데이터 재매칭

데이터 재매칭은 썸네일 재생성, 메타데이터 큐 등록, 해시 재생성, DB 참조 리매칭을 실행합니다. 해시 재생성은 참조 관계에 영향이 크므로 필요한 경우에만 사용하세요.

## 감시 폴더 탭

주요 작업:

- 감시 폴더 추가/수정/삭제
- 백업 소스 추가/수정/삭제
- 전체 스캔
- 폴더별 스캔
- watcher 시작/중지/재시작
- 전체 파일 검증
- 최근 스캔 로그 확인

자세한 내용은 [감시 폴더와 백업 소스](./WATCHED_FOLDERS.md)를 참고하세요.

## 보안 탭

주요 작업:

- 인증 상태 확인
- 첫 계정 설정
- 현재 계정 정보 변경
- 계정 목록 관리
- 계정별 권한 그룹 지정
- 권한 그룹 생성/수정
- 페이지 권한 설정
- 권한 그룹 색상 설정
- 복구 정보 확인

외부 접속을 열기 전에는 anonymous, guest, admin 권한을 반드시 확인하세요.

## 자동 탭

주요 작업:

- Kaloscope 설정과 상태 확인
- WD Tagger 설정과 의존성 확인
- 평가 가중치 설정
- 평가 등급 설정
- 해시 기준 테스트 이미지 확인
- 랜덤 이미지로 자동 추출 테스트

업로드 페이지의 자동 추출/작가 추출과 이미지 상세의 자동/작가 프롬프트 표시에 연결됩니다.

## 메타데이터 탭

주요 항목:

- PNG secondary extraction
- Stealth 스캔 모드: 빠르게 / 전체 / 건너뛰기
- Stealth 최대 파일 크기
- Stealth 최대 해상도
- ComfyUI/WebUI 판별 시 fallback 스킵
- 모든 항목 재추출

전체 재추출은 기존 이미지들을 다시 큐에 넣는 작업입니다. 대량 라이브러리에서는 시간이 걸릴 수 있습니다.

## 이미지 저장 탭

주요 항목:

- 생성 텀/쓰로틀: 예약작업, NovelAI, Codex 동시 실행 수와 cooldown
- 기본 저장 포맷: 원본 유지, PNG, JPEG, WebP
- 품질
- 리사이즈와 최대 크기
- 매번 팝업 확인 여부
- 생성 첨부/에디터/캔버스/업로드/워크플로 출력 적용 여부
- 비디오 최적화

생성 큐가 너무 빠르게 몰리거나 외부 서비스 제한이 걱정되면 생성 텀 값을 보수적으로 잡습니다.

## 변경 후 확인 순서

1. 설정 저장 버튼을 눌렀는지 확인합니다.
2. 관련 화면을 새로고침합니다.
3. 스캔/추출/생성처럼 큐가 필요한 작업은 로그나 상태를 확인합니다.
4. 권한 변경은 다른 계정 또는 `/access`에서 확인합니다.
5. 저장 경로/삭제 보호 변경은 테스트 파일 하나로 먼저 확인합니다.
