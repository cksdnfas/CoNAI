# 이미지 상세와 유사도 검사

이미지 상세는 한 파일의 미리보기, 메타데이터, 프롬프트, 유사 이미지를 확인하는 화면입니다.

## 관련 페이지

| 목적 | 경로 |
| --- | --- |
| 상세 페이지 | `/images/:compositeHash` |
| 메타데이터 편집 | `/images/:compositeHash/metadata` |
| 홈 피드 | `/` |
| 유사도 기본 정책 | `/settings` → 일반 설정 |

`compositeHash`는 CoNAI가 파일을 식별할 때 사용하는 복합 해시입니다. 홈, 업로드 결과, 그룹, 생성 이력에서 상세로 이동할 수 있습니다.

## 상세 화면에서 보는 것

| 영역 | 설명 |
| --- | --- |
| 미디어 미리보기 | 이미지/비디오를 크게 표시합니다. |
| 메타데이터 | 복합 해시, 크기, 파일 크기, 모델, 파라미터, 원본 경로를 표시합니다. |
| 추출 프롬프트 | 긍정/부정 프롬프트를 표시하고 복사할 수 있습니다. |
| 자동 프롬프트 | 자동 태그 결과를 등급/캐릭터/일반 태그로 나눠 보여줍니다. |
| 작가 프롬프트 | Kaloscope 작가 후보와 링크를 표시합니다. |
| 중복 이미지 | 같은/매우 가까운 해시를 가진 항목을 보여줍니다. |
| 유사 이미지 | 이미지 해시·색상 기준 유사 항목을 보여줍니다. |
| 텍스트 유사 | 프롬프트 텍스트 기준 유사 항목을 보여줍니다. |

## 상단 작업 버튼

| 버튼 | 기능 |
| --- | --- |
| 뒤로 | 홈 또는 이전 상세 진입 지점으로 돌아갑니다. |
| 새로고침 | 현재 상세 데이터를 다시 불러옵니다. |
| 메타데이터 편집 | `/images/:compositeHash/metadata`로 이동합니다. 정적 이미지에서만 사용합니다. |
| 그룹 추가 | 현재 이미지를 커스텀 그룹에 추가합니다. |
| 다운로드 | 원본 또는 렌더 가능한 파일을 내려받습니다. |

## 프롬프트 표시 방식

추출 프롬프트에는 두 가지 표시 방식이 있습니다.

- **Plain**: 원본에 가까운 프롬프트 텍스트를 그대로 표시합니다.
- **Group**: CoNAI 프롬프트 그룹 기준으로 태그를 묶어 표시합니다.

Group 표시에서는 분류 깊이를 조절하거나 Danbooru를 루트 그룹처럼 취급하는 옵션을 사용할 수 있습니다. 이 값은 브라우저 로컬 설정으로 저장됩니다.

## 자동 프롬프트와 작가 프롬프트

자동 프롬프트 영역은 자동 태그 결과를 다음처럼 분리합니다.

- Rating
- Character
- General tags

일반 태그는 Danbooru 링크로 열 수 있으며, 복사 버튼으로 자동 태그 텍스트를 가져올 수 있습니다.

작가 프롬프트 영역은 Kaloscope 결과를 표시합니다. 작가 링크 템플릿은 상세 화면의 작가 설정 버튼 또는 `/settings` → 자동 설정에서 조정합니다.

## 유사/중복 검사 실행

유사도 검사는 두 방식으로 실행됩니다.

| 방식 | 설명 |
| --- | --- |
| 수동 | 상세 화면에서 **유사/중복 검사** 버튼을 누릅니다. |
| 자동 | `/settings` → 일반 설정의 유사/중복 검사 정책이 자동 실행이면 상세 열 때 실행됩니다. |

검사는 이미지 로드가 안정화된 뒤 실행됩니다. 정적 이미지가 아닌 비디오/애니메이션 항목은 일부 유사도 영역이 제한될 수 있습니다.

## 유사도 탭

| 탭 | 기준 |
| --- | --- |
| 이미지 | perceptual hash, dHash, aHash, 색상 유사도 옵션 기준 |
| 텍스트 | 긍정/부정/자동 프롬프트의 텍스트 유사도 기준 |

설정 권한이 있는 계정은 상세 화면의 유사도 설정 패널에서 임계값, 결과 수, 가중치, 메타데이터 필터, 프롬프트 유사도 가중치를 바로 조정할 수 있습니다.

## 점수 해석

유사 카드에는 점수 배지가 표시됩니다.

- 중복 이미지는 중복/해시 기준으로 먼저 분리됩니다.
- 이미지 유사도는 설정한 해시·색상 가중치에 따라 달라집니다.
- 텍스트 유사도는 프롬프트 필드별 가중치와 임계값에 따라 달라집니다.

결과가 너무 많으면 임계값을 높이거나 결과 수를 줄입니다. 결과가 너무 적으면 임계값을 낮추고 색상/프롬프트 가중치를 조정합니다.

## 상세이 비어 있거나 느릴 때

1. 원본 파일 경로가 존재하는지 확인합니다.
2. 썸네일/렌더 파일이 아직 처리 중인지 확인합니다.
3. 새로고침 버튼을 눌러 DB 상태를 다시 불러옵니다.
4. 대량 파일 스캔 중이면 처리가 끝날 때까지 기다립니다.
5. 권한 때문에 상세 접근이 막히는지 `/access`에서 확인합니다.
