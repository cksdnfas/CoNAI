# NAI 생성

NAI 탭은 NovelAI 계정으로 이미지를 생성하고, 결과를 CoNAI 생성 이력에 저장하는 화면입니다.

## 위치

`/generation?tab=nai`

## 기본 흐름

1. NAI 탭을 엽니다.
2. 로그인 버튼으로 NovelAI 계정을 연결합니다.
3. 프롬프트와 네거티브 프롬프트를 입력합니다.
4. 모델, 액션, 샘플링, 해상도, 샘플 수를 조정합니다.
5. 필요하면 캐릭터 프롬프트, 캐릭터 레퍼런스, Vibe 이미지를 추가합니다.
6. 생성 버튼을 누릅니다.
7. 오른쪽 생성 이력에서 결과와 실패 메시지를 확인합니다.

## 로그인과 비용

상단 연결 영역에서 로그인 상태, 구독 티어, Anlas 잔액을 확인합니다.

생성 버튼에는 가능한 경우 예상 비용이 표시됩니다.

- Opus 무료 조건이면 무료 생성으로 표시됩니다.
- 비용 계산 중이면 계산 상태가 표시됩니다.
- 로그인 전에는 생성 버튼이 비활성화됩니다.

## 프롬프트

| 항목 | 용도 |
| --- | --- |
| Prompt | 만들고 싶은 이미지 설명 |
| Negative Prompt | 피하고 싶은 요소 |
| Character Prompt | 캐릭터별 프롬프트/UC 분리 |
| AI's Choice | 캐릭터 위치를 NAI에 맡김 |

캐릭터 위치를 직접 쓰면 보드에서 각 캐릭터의 위치를 지정할 수 있습니다. 지원하지 않는 모델에서는 캐릭터 프롬프트/위치 옵션이 비활성화됩니다.

## 주요 설정

| 설정 | 설명 |
| --- | --- |
| Model | NAI Diffusion 모델 선택 |
| Action | Generate, img2img, infill 등 작업 선택 |
| Sampler / Scheduler | 샘플링 방식과 스케줄러 |
| Steps | 생성 반복 수 |
| CFG Scale | 프롬프트 반영 강도 |
| Preset / Width / Height | 해상도 프리셋 또는 직접 입력 |
| Samples | 한 번에 생성할 이미지 수. 1~4 범위로 제한됩니다. |
| Seed | 재현용 시드. 비우면 랜덤입니다. |
| Variety+ | 지원 모델에서 변주 옵션을 사용합니다. |

## 이미지 작업

Action이 `generate`가 아니면 이미지 입력 영역이 열립니다.

| 작업 | 필요한 입력 |
| --- | --- |
| img2img | Source Image |
| infill | Source Image + Mask Image |

Source Image는 기존 이미지 기반 생성에 사용합니다.

Mask Image는 인페인트 영역을 지정합니다. 마스크는 내장 이미지 편집기에서 만들거나 별도 파일로 선택할 수 있습니다.

추가 옵션:

- Strength: 원본 이미지 유지/변형 강도
- Noise: 이미지 변형 노이즈
- Original: infill에서 원본 보존 옵션

## 레퍼런스와 Vibe

### Character Reference

캐릭터 외형을 참고할 이미지를 추가합니다.

- 이미지 선택
- 타입 선택
- Strength 조정
- Fidelity 조정
- 자주 쓰는 레퍼런스 저장/불러오기

지원 모델에서만 활성화됩니다.

### Vibe

스타일·분위기를 참고할 이미지를 추가합니다.

- 이미지 선택
- Strength 조정
- Information Extracted 조정
- 저장된 Vibe 불러오기

Vibe 이미지는 생성 전 인코딩이 필요할 수 있습니다. 연결 상태가 끊기면 먼저 로그인 상태를 확인합니다.

## 결과 확인

생성 후 결과는 생성 이력 패널에 표시됩니다.

확인할 것:

- 성공/실패 상태
- 결과 이미지
- 저장 포맷과 리사이즈 적용 여부
- 재시도 가능 여부
- 업스케일 가능 여부

## 빠른 팁

- 일반 생성은 Prompt, Negative Prompt, Model, 해상도만 맞춰도 충분합니다.
- 여러 캐릭터는 Character Prompt로 분리합니다.
- 원본 이미지가 중요하면 Strength를 낮게 시작합니다.
- 재현이 필요하면 Seed를 고정합니다.
- 결과 저장 형식은 설정 → 미디어 생성/저장에서 관리합니다.
