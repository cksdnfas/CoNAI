# 그룹 관리

그룹은 이미지를 폴더 구조나 사용자가 정한 기준으로 묶는 기능입니다. CoNAI에는 커스텀 그룹과 감시폴더 그룹이 있습니다.

## 관련 페이지

| 목적 | 경로 |
| --- | --- |
| 커스텀 그룹 | `/groups` |
| 감시폴더 그룹 | `/groups?tab=folders` |
| 특정 그룹 | `/groups/:groupId` |
| 감시 폴더 설정 | `/settings` → 감시 폴더 |

## 그룹 종류

| 종류 | 설명 | 직접 편집 |
| --- | --- | --- |
| 커스텀 그룹 | 사용자가 만든 이미지 묶음입니다. 즐겨찾기, 작업별 모음, 캐릭터별 모음에 적합합니다. | 가능 |
| 감시폴더 그룹 | 감시 폴더 구조를 기준으로 자동 구성되는 그룹입니다. | 제한적 |

커스텀 그룹은 사용자가 생성/수정/삭제할 수 있습니다. 감시폴더 그룹은 등록된 폴더와 스캔 결과에서 만들어지며, 루트에서 재구성할 수 있습니다.

## 기본 화면

`/groups`는 왼쪽 탐색 패널과 오른쪽 콘텐츠 영역으로 구성됩니다.

- 상단 탭: 커스텀 그룹 / 감시폴더 그룹
- 탐색 패널: 전체 그룹 계층
- 루트 화면: 하위 그룹 카드
- 그룹 상세: 그룹 정보, 하위 그룹, 이미지 목록
- 하단 선택 바: 선택 이미지 작업

이미지 목록의 열 수는 그룹 페이지 전용 컨트롤로 조정합니다.

## 커스텀 그룹 만들기

1. `/groups`에서 **커스텀 그룹** 탭을 엽니다.
2. 새 그룹 버튼을 누릅니다.
3. 그룹 이름을 입력합니다.
4. 필요하면 설명, 부모 그룹, 강조 색상을 지정합니다.
5. 자동수집을 쓰려면 **필터 적용**을 켭니다.
6. 저장합니다.

자동수집 조건은 칩 편집기 또는 JSON 직접 편집으로 만들 수 있습니다.

## 자동수집 조건

자동수집은 이미지의 검색 조건을 그룹 규칙으로 저장하는 기능입니다.

칩 편집기에서 사용할 수 있는 대표 범위:

- 긍정 프롬프트
- 부정 프롬프트
- 자동 태그
- 모델명
- LoRA 이름
- 등급
- AI Tool

조건 칩의 연산자는 칩에서 순환 변경할 수 있습니다. 기존 규칙이 칩 편집기로 표현하기 어렵다면 JSON 직접 편집 모드로 열립니다.

## 이미지 넣기

이미지를 커스텀 그룹에 넣는 방법은 두 가지입니다.

1. 홈 또는 그룹 이미지 목록에서 여러 이미지를 선택합니다.
2. 선택 바의 **그룹 추가**를 누릅니다.
3. 대상 커스텀 그룹을 선택합니다.
4. 저장합니다.

또는 이미지 상세 화면에서 현재 이미지만 그룹에 추가할 수 있습니다.

## 그룹 상세에서 할 수 있는 일

| 작업 | 설명 |
| --- | --- |
| 그룹 열기 | 탐색 패널이나 카드에서 하위 그룹으로 이동합니다. |
| 상위로 이동 | breadcrumbs 또는 뒤로 카드로 상위 그룹으로 이동합니다. |
| 그룹 다운로드 | 현재 그룹의 원본/썸네일/비디오를 아카이브로 받습니다. |
| 선택 다운로드 | 선택한 이미지 기준으로 내려받습니다. |
| 그룹 편집 | 이름, 설명, 부모, 색상, 자동수집 조건을 수정합니다. |
| 자동수집 실행 | 현재 그룹 조건으로 이미지를 다시 모읍니다. |
| 선택 제거 | 커스텀 그룹에서 선택 이미지를 제외합니다. 원본 파일 삭제가 아닙니다. |
| 선택 삭제 | 삭제 권한이 있을 때 실제 이미지 삭제 흐름을 실행합니다. |

## 감시폴더 그룹 재구성

감시폴더 그룹 탭의 루트에서 **감시폴더 재구성**을 실행하면 현재 감시 폴더와 파일 경로 기준으로 자동 그룹 계층을 다시 만듭니다.

사용 시점:

- 감시 폴더를 새로 등록했을 때
- 폴더 구조가 크게 바뀌었을 때
- 감시폴더 그룹 수가 실제 폴더 구조와 맞지 않을 때

## 다운로드 옵션

그룹 다운로드 모달은 받을 파일 종류를 선택합니다.

- 원본 이미지
- 썸네일
- 비디오/애니메이션 파일

그룹에 실제 파일이 없거나 원본 경로가 누락된 항목은 다운로드 수에 포함되지 않을 수 있습니다.

## 운영 팁

- 감시폴더 그룹은 원본 폴더 구조를 보는 용도로 사용합니다.
- 작업 큐, 즐겨찾기, 품질 선별은 커스텀 그룹으로 관리합니다.
- 자동수집 조건은 처음엔 좁게 만들고 결과를 확인한 뒤 확장합니다.
- 삭제와 그룹 제거는 다릅니다. 그룹 제거는 그룹 관계만 빼고, 삭제는 파일/DB 삭제 흐름입니다.
