# 업로드와 메타데이터 추출

업로드 페이지는 파일을 CoNAI에 직접 등록하거나, 등록하지 않고 단일 이미지의 메타데이터만 확인하는 작업 공간입니다.

## 관련 페이지

| 목적 | 경로 |
| --- | --- |
| 업로드/추출 | `/upload` |
| 이미지 상세 | `/images/:compositeHash` |
| 저장 정책 설정 | `/settings` → 이미지 저장 |
| 메타데이터 추출 설정 | `/settings` → 메타데이터 |

## 화면 구성

`/upload`에는 두 개의 주요 섹션이 있습니다.

| 섹션 | 용도 |
| --- | --- |
| 파일 업로드 | 여러 이미지/비디오를 CoNAI 라이브러리에 저장합니다. |
| 미리보기/추출 | 단일 이미지의 프롬프트/태그/작가 후보를 확인하고 파일을 변환합니다. |

## 업로드 지원 파일

파일 업로드 영역은 여러 파일을 한 번에 받을 수 있습니다.

- 이미지: JPEG, PNG, WebP, TIFF, BMP, GIF
- 비디오: MP4, WebM, QuickTime/MOV, AVI, MKV

업로드 후 저장 옵션 모달이 열리면 현재 저장 정책을 확인하고 저장을 확정합니다. 저장 정책은 `/settings` → 이미지 저장에서 관리합니다.

## 업로드 절차

1. `/upload`를 엽니다.
2. 파일 업로드 영역을 클릭하거나 파일을 끌어다 놓습니다.
3. 선택된 파일 수와 총 용량을 확인합니다.
4. **업로드**를 누릅니다.
5. 저장 옵션이 표시되면 보관 방식과 출력 옵션을 확인합니다.
6. 진행률이 100%가 될 때까지 기다립니다.
7. 결과 목록에서 성공/실패 수를 확인합니다.
8. 성공 항목의 상세 열기 버튼으로 상세 페이지를 확인합니다.

## 업로드 결과 읽기

| 표시 | 의미 |
| --- | --- |
| 성공 | 저장과 DB 등록이 끝난 파일 수 |
| 실패 | 포맷, 경로, 저장, 메타데이터 처리 중 실패한 파일 수 |
| 상세 열기 | 해당 파일의 `/images/:compositeHash` 화면으로 이동 |
| 진행률 | 전송된 바이트와 전체 바이트 기준 진행 상태 |

실패가 섞여 있어도 성공한 파일은 정상 등록될 수 있습니다. 실패 파일은 오류 메시지를 기준으로 다시 업로드하거나 원본 파일을 확인합니다.

## 미리보기/추출 영역

미리보기/추출은 **단일 이미지** 기준입니다. 파일을 선택하면 먼저 내장 메타데이터 프롬프트 추출을 시도합니다.

표시 항목:

- 파일명, 크기, MIME 타입
- 이미지 크기
- AI 도구
- 모델명
- 스텝, 샘플러, 시드 등 생성 파라미터
- LoRA 목록
- 긍정/부정 프롬프트
- 자동 태그 결과
- 작가 후보 결과

## 추출 작업

| 작업 | 설명 |
| --- | --- |
| 추출 전체 | 내장 프롬프트, 자동 태그, 작가 후보를 함께 실행합니다. |
| 자동 추출 | WD Tagger 계열 자동 태그 결과를 확인합니다. |
| 작가 추출 | Kaloscope 계열 작가 후보 결과를 확인합니다. |
| WebP 변환 | 선택 이미지를 WebP로 변환해 다운로드합니다. |
| 메타데이터 편집 | 프롬프트/모델/스텝/샘플러 값을 수정한 파일을 다운로드합니다. |

자동/작가 추출은 관련 모델과 설정 상태에 영향을 받습니다. 실패하면 `/settings` → 자동 또는 `/settings` → 메타데이터를 확인하세요.

## 미리보기에서 메타데이터 수정 다운로드

1. 미리보기/추출 영역에 이미지를 넣습니다.
2. **메타데이터 편집** 패널을 펼칩니다.
3. 출력 형식(PNG/JPEG/WebP), 프롬프트, 네거티브 프롬프트, 모델, 스텝, 샘플러를 수정합니다.
4. 상단의 **메타데이터 편집** 다운로드 버튼을 누릅니다.
5. 수정된 메타데이터가 들어간 새 파일을 내려받습니다.

이 작업은 업로드된 DB 항목을 바꾸지 않고 새 파일을 다운로드합니다. 이미 등록된 이미지의 메타데이터를 DB/파일 기준으로 저장하려면 [이미지 메타데이터 편집](./EDIT_IMAGE_METADATA.md)을 사용하세요.

## 문제 해결

| 증상 | 확인할 것 |
| --- | --- |
| 업로드 실패 | 파일 포맷, 파일 잠금, 저장 경로, 디스크 여유 공간 |
| 메타데이터가 비어 있음 | 원본 파일에 AI 메타데이터가 실제로 있는지 확인 |
| 자동 추출 실패 | `/settings` → 자동의 Tagger 상태와 모델 설정 |
| 작가 추출 실패 | `/settings` → 자동의 Kaloscope 상태와 모델 설정 |
| WebP 변환 실패 | 원본 파일 손상 여부와 브라우저 다운로드 차단 여부 |
| 상세 링크가 없음 | 업로드 결과가 이미지/비디오 레코드로 등록되었는지 확인 |
