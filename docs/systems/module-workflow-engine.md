# 워크플로우 실행 엔진

## 역할

CoNAI의 Workflow 기능을 담당하는 중심 시스템입니다.
그래프 편집, 실행, 재실행, 실행 기록, 결과물 관리가 여기서 이어집니다.

## 이 시스템이 하는 일

- 워크플로우 그래프를 저장하고 불러옵니다.
- 노드와 엣지 연결 상태를 검증합니다.
- 워크플로우 실행과 노드 단위 실행을 처리합니다.
- 실행 이력과 결과물을 조회합니다.
- 폴더 단위로 워크플로우를 정리하고 브라우즈합니다.

## 연결되는 대표 화면

- Image Generation > Workflow 탭
- `/graph` 리다이렉트 경로
- 워크플로우 브라우즈/결과물 관리 패널

## 사용자 입장에서 보이는 효과

- 시각적으로 생성 흐름을 구성할 수 있습니다.
- 실행 이력과 산출물을 추적할 수 있습니다.
- 향후 생성 파이프라인의 고급 자동화 기반이 됩니다.

## 문서화 포인트

- 단순 UI 캔버스가 아니라 실행 시스템입니다.
- 편집 모드와 브라우즈 모드가 나뉩니다.
- 결과물 관리는 생성 기능과 별도 설명이 필요합니다.
