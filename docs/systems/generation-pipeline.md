# 생성 저장 파이프라인

## 역할

생성 결과물을 어떻게 저장하고, 어떤 포맷으로 내보내고, 운영 파일을 어떻게 보존할지 결정하는 공통 축입니다.
화면마다 제각각 저장되면 금방 지저분해지니까, 이 파이프라인이 질서를 잡습니다.

## 이 시스템이 하는 일

- 생성 결과 저장 옵션을 공통 구조로 받습니다.
- 포맷, 품질, 리사이즈 정책을 반영합니다.
- 다운로드용 파일 재작성과 운영 저장을 분리합니다.
- 메타 수정 저장 시 기존 운영 파일 보존 전략과 연결됩니다.
- RecycleBin 기반 보존 흐름을 지원합니다.

## 중요한 문서화 포인트

- NAI, ComfyUI, Upload, Metadata Edit가 모두 이 축과 연결됩니다.
- 단순 다운로드와 실제 운영 파일 저장은 서로 다른 책임입니다.
- 저장 정책은 Settings의 image-save 설정과 강하게 연결됩니다.

## 연결되는 대표 화면

- Image Generation
- Upload
- Metadata Edit
- Settings > Image Save

## 사용자 입장에서 보이는 효과

- 생성물 저장 방식이 통일됩니다.
- 포맷과 품질 설정이 실제 결과에 반영됩니다.
- 메타 수정 시 운영 파일이 무작정 덮어쓰기되지 않습니다.
