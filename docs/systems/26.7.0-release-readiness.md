# 26.7.0 release readiness

이 문서는 26.7.0 workflow-operations upgrade를 로컬에서 검토 가능한 상태로 묶는 체크리스트입니다. 실제 push, deploy, tag, live demo update는 포함하지 않습니다.

## 목표

- 재사용 가능한 workflow authoring 경로를 문서에서 바로 설명할 수 있다.
- graph execution, queue, artifact, final result 확인 흐름을 알파 리뷰 기준으로 설명할 수 있다.
- generated media를 search, group, similarity, recycle/delete lock 관점에서 검토할 수 있다.
- MCP/demo 표면은 opt-in과 trusted-client 전제로 설명한다.

## 로컬 검증

릴리즈 준비 alias:

```bash
npm run verify:release-readiness
```

Alias 구성:

1. `npm run verify:workspace-script-aliases`
2. `npm run docs:build`
3. `npm run build`

Graphify가 있는 로컬 작업에서는 코드 변경 후 다음도 실행합니다.

```bash
python -m graphify update .
```

## 알파 리뷰 확인 항목

- Workflow authoring: module library grouping, Comfy workflow entry, random/API/bypass nodes
- Execution operations: queue lane/wait/duration, split API/worker roles, final result and artifact inspection
- Media organization: search suggestions, detail prefetch, auto-tag/group rematch, similarity, thumbnail defaults, recycle/delete locks
- Demo/MCP: `CONAI_MCP_HTTP_ENABLED=true` opt-in, wildcard guest permissions, anonymous demo route guards, trusted-client MCP guide

## 승인 필요 항목

다음은 이 readiness package의 범위를 넘습니다.

- Git push, release tag, public announcement
- live demo host pull, deploy, service restart
- protected service `3999` 조작
- auth/security policy redesign
- DB schema, retention policy, destructive cleanup
- external credential, billing, cloud sync, or public API expansion
