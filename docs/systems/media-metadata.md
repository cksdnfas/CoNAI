# 미디어 메타데이터 엔진

## 역할

CoNAI의 검색, 정렬, 필터링, 프롬프트 분석은 전부 이 계층을 중심으로 돌아갑니다.
파일 자체를 직접 만지지 않아도 대부분의 조회 작업을 수행할 수 있게 해주는 핵심 엔진입니다.

## 이 시스템이 하는 일

- composite hash 기준으로 미디어를 식별합니다.
- 이미지와 비디오의 메타데이터를 저장합니다.
- 프롬프트, 모델명, 태그, 생성 파라미터를 검색 가능한 구조로 유지합니다.
- prompt similarity용 정규화 필드와 fingerprint를 함께 관리합니다.
- 페이지에서 쓰는 목록 조회와 상세 조회의 기준 데이터를 제공합니다.

## 연결되는 대표 화면

- Home
- Groups
- Prompts
- Image Detail
- Metadata Edit

## 사용자 입장에서 보이는 효과

- 검색 결과가 빠르게 나옵니다.
- 프롬프트 관련 정리 기능이 가능한 이유입니다.
- 이미지 상세 페이지에서 메타를 안정적으로 읽고 수정할 수 있습니다.

## 문서화 포인트

- 단순 DB 테이블이 아니라, CoNAI의 조회 중심 도메인 모델입니다.
- 파일과 메타는 분리되지만 composite hash로 연결됩니다.
- 메타 수정은 곧 검색성과 프롬프트 분석 결과에도 영향을 줍니다.
