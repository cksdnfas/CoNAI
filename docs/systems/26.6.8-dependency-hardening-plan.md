# 26.6.8 dependency hardening plan

이 문서는 `26.6.8 hardening-evidence-observability` 로드맵의 M1-CU1 결과입니다. 2026-06-08 KST 기준 `npm audit`를 다시 실행해 취약 패키지를 기능 표면과 패키지 루트에 매핑하고, 다음 커밋에서 적용할 수 있는 안전한 업데이트와 승인 없이 진행하면 안 되는 후보를 분리합니다.

## 감사 기준

실행한 점검:

- `npm audit`
- `npm audit --omit=dev`
- `npm audit fix --dry-run --json`
- `npm outdated --json`
- `npm explain`으로 직접/전이 의존성 경로 확인
- 코드 검색으로 실제 사용 표면 확인

결과 요약:

- 전체 `npm audit`: moderate 11건, high 11건, 총 22건.
- 런타임 중심 `npm audit --omit=dev`: moderate 7건, high 8건, 총 15건.
- `npm audit fix --dry-run --json`: `vite@8.0.1`의 optional peer `esbuild@"^0.27.0"`와 루트 직접 devDependency `esbuild@0.25.12` 충돌로 `ERESOLVE` 중단. 자동 수정은 안전한 경로로 보지 않습니다.

## 패키지 루트와 영향 표면

### Backend runtime and API

- `axios@1.13.6`는 `backend/package.json`의 직접 dependency입니다. 실제 사용 표면은 NovelAI auth/generate/upscale, ComfyUI HTTP 호출, MCP NovelAI generation tool입니다.
- `express-rate-limit@8.3.1`는 backend 직접 dependency이며 `ip-address@10.1.0` 취약점과 연결됩니다. 실제 사용 표면은 login, API, upload, read-only rate limiter입니다.
- `express@5.2.1`의 전이 의존성 `router -> path-to-regexp@8.3.0`, `body-parser/express -> qs@6.15.0`가 audit에 잡힙니다. 모든 backend route matching, body parsing, static serving 표면과 연결됩니다.
- `@modelcontextprotocol/sdk@1.27.1`는 backend 직접 dependency이며 `hono`, `@hono/node-server`, `ajv`, `fast-uri` 취약점의 주요 루트입니다. 실제 사용 표면은 backend MCP HTTP/stdio 서버와 generation/image/prompt/resource MCP tools입니다.
- `better-queue@3.8.12`와 `uuid@13.0.0`는 backend 직접 dependencies입니다. 실제 사용 표면은 legacy job tracker/temp image identifiers와 background queue dependency chain입니다.

### Frontend runtime

- `react-router-dom@7.13.1`는 frontend 직접 dependency이며 `react-router@7.13.1` 취약점과 연결됩니다. 실제 사용 표면은 hash router, protected route redirect, navigation links, detail/settings/group/workflow pages입니다.

### Build, docs, and development tooling

- `vite@8.0.1`는 frontend 직접 devDependency이며 frontend build/dev server 표면입니다. 연결 취약점은 Vite dev-server 파일 접근 계열과 `postcss@8.5.8`입니다.
- `vitepress@1.6.4`는 root 및 docs devDependency이며 docs build/dev server 표면입니다. `vitepress -> vite@5.4.21 -> esbuild@0.21.5` 계열은 audit에서 `fixAvailable: false`로 표시됩니다.
- `eslint`, `shadcn`, `@modelcontextprotocol/sdk`, `vitepress` 주변 dev tooling에서 `flatted`, `minimatch`, `brace-expansion`, `picomatch`, `postcss`, `ajv`, `fast-uri`가 전이 취약점으로 잡힙니다.
- `shared/package.json` 자체에서는 직접 취약 dependency가 확인되지 않았습니다.

## CU2 안전 업데이트 후보

다음은 같은 major 또는 명확한 minor/patch 범위로 보이며, M1-CU2에서 package manifest/lock 변경 후 focused verification을 통과하면 안전 lane으로 볼 수 있습니다.

1. `axios` `1.13.6 -> 1.17.0`
   - 이유: backend 직접 dependency이며 `npm outdated`의 wanted/latest가 `1.17.0`.
   - 확인 표면: NovelAI auth/generate/upscale, ComfyUI request/queue/system_stats, MCP NovelAI tool.
   - 권장 확인: `npm audit`, `npm run build`, `npm run verify:codex-generation-contracts`, `npm run verify:mcp-generation-tools-contracts`, 관련 ComfyUI route contract 중 변경 영향이 있는 항목.

2. `express-rate-limit` `8.3.1 -> 8.5.2`
   - 이유: backend 직접 dependency이며 `ip-address` 전이 취약점을 같이 줄일 가능성이 큼.
   - 확인 표면: login/API/upload/read-only limiter와 Express `trust proxy` 설정.
   - 권장 확인: `npm audit`, `npm run build`, `npm run verify:auth-db-bootstrap`, `npm run verify:nai-user-data-contracts`.

3. `react-router-dom` `7.13.1 -> 7.17.0`
   - 이유: frontend 직접 dependency이며 `react-router` high 취약점의 루트.
   - 확인 표면: `createHashRouter`, protected route redirect, `useBlocker`, route error boundary, detail/settings/group/workflow navigation.
   - 권장 확인: `npm audit`, `npm run build`, `npm run verify:release-readiness-workspace`, `npm run verify:integrated-operations-surface-contracts`.

4. `uuid` `13.0.0 -> 13.0.2`
   - 이유: backend 직접 dependency의 same-major patch 후보.
   - 확인 표면: `jobTracker.ts`, `tempImageService.ts`.
   - 권장 확인: `npm audit`, `npm run build`, focused backend script checks around temporary image/job IDs if touched.

5. `@modelcontextprotocol/sdk` `1.27.1 -> 1.29.0`
   - 이유: backend 직접 dependency이고 `hono`, `@hono/node-server`, `ajv`, `fast-uri` transitive risk의 주 루트.
   - 확인 표면: backend MCP HTTP/stdio transport and tool registration.
   - 권장 확인: `npm audit`, `npm run build`, `npm run verify:mcp-generation-tools-contracts`, `npm run verify:runtime-role-contracts`.

## 승인 또는 별도 설계가 필요한 후보

- `npm audit fix --force` 또는 자동 downgrade/override는 승인 없이 진행하지 않습니다. 현재 dry-run도 `ERESOLVE`로 중단됩니다.
- `better-queue` audit fix는 `better-queue@3.8.4`를 제안하지만 현재 직접 범위 `^3.8.12`보다 낮은 버전입니다. queue runtime에 걸린 dependency이므로 downgrade, replacement, nested `uuid` override는 M1-CU2의 안전 lane으로 단정하지 않습니다.
- `vite`, root `esbuild`, `vitepress` 조합은 dev/docs tooling이지만 peer 충돌과 `fixAvailable: false`가 있습니다. Vite만 올리는 방식, root `esbuild`를 `^0.27`로 올리는 방식, VitePress nested Vite를 override하는 방식은 각각 docs/build verification 뒤에만 판단해야 합니다.
- `express -> router -> path-to-regexp`와 `express/body-parser -> qs`는 backend route/body parsing에 걸립니다. patch-level transitive 해결이 가능하면 안전 lane이지만, Express major 변경이나 router override가 필요하면 approval-needed로 돌립니다.
- `eslint`, `shadcn`, `vitepress` 전이 dev dependency 묶음은 release runtime보다 tooling risk가 큽니다. broad dev-toolchain rewrite, lint config migration, docs generator migration은 현재 커밋 유닛 범위를 넘습니다.

## M1-CU2 작업 순서 제안

1. 먼저 runtime high-risk direct roots를 최소 묶음으로 업데이트합니다: `axios`, `express-rate-limit`, `react-router-dom`, direct `uuid`.
2. package lock consistency를 확인한 뒤 `@modelcontextprotocol/sdk` minor update를 별도 묶음으로 적용합니다.
3. `npm audit` 결과에서 남은 runtime transitive 항목이 Express/router/qs 또는 better-queue 계열인지 확인합니다.
4. dev/docs-only 항목은 `npm audit --omit=dev`가 깨끗해진 뒤 별도 판단합니다.
5. 자동 `npm audit fix --force`, downgrade, broad override, package version bump, git tag, push, deploy, restart는 실행하지 않습니다.

## 현재 남은 위험

- 런타임 취약점 15건은 아직 적용되지 않았습니다. 이 커밋은 감사/분류와 안전 lane 정의만 수행합니다.
- `axios` 계열은 external HTTP call 표면이 넓어 build만으로 충분하지 않습니다. NovelAI, ComfyUI, MCP generation 경로의 focused contract/smoke가 필요합니다.
- `@modelcontextprotocol/sdk` 업데이트는 MCP transport behavior와 tool schema behavior를 바꿀 수 있습니다.
- `better-queue`는 legacy queue dependency이지만 audit fix가 downgrade를 제안하므로 replacement/override 판단 전에는 직접 수정하지 않는 편이 안전합니다.
- dev/docs tooling 취약점은 local dev server exposure risk입니다. 릴리즈 runtime risk와 분리해 다뤄야 합니다.

## M1-CU2 적용 결과

2026-06-08 KST M1-CU2에서는 승인 없이 처리 가능한 same-major/minor/patch dependency hardening만 적용했습니다.

적용한 직접 dependency:

- backend: `axios` `^1.13.6 -> ^1.17.0`
- backend: `express-rate-limit` `^8.3.1 -> ^8.5.2`
- backend: `@modelcontextprotocol/sdk` `^1.27.1 -> ^1.29.0`
- backend: `uuid` `^13.0.0 -> ^13.0.2`
- frontend: `react-router-dom` `^7.13.1 -> ^7.17.0`
- frontend/tooling: `vite` `^8.0.1 -> ^8.0.16`
- root tooling: `esbuild` `^0.25.12 -> ^0.27.7`, to satisfy Vite 8's peer lane before running non-force audit repair.

추가 lockfile hardening:

- `npm audit fix` without `--force` resolved safe transitive advisories for MCP/Hono/AJV/Fast URI, Express router/query parsing, frontend Vite transitive tooling, and common build/lint globs.
- `npm audit` moved from 22 vulnerabilities (11 moderate, 11 high) to 5 moderate vulnerabilities, with no high vulnerabilities remaining.
- `npm audit --omit=dev` moved from 15 runtime vulnerabilities to 2 moderate runtime vulnerabilities, both tied to `better-queue -> uuid`.

남은 승인/설계 후보:

- `better-queue@3.8.12 -> better-queue@3.8.4` is the only npm-proposed runtime fix for the nested `uuid` advisory, but it is a downgrade/breaking audit path and remains out of scope without approval or replacement design.
- `vitepress@1.6.4 -> vite@5.4.21 -> esbuild@0.21.5` remains a docs/dev-server advisory with `fixAvailable: false`; handling it needs a VitePress/docs toolchain decision rather than a forced audit repair.
