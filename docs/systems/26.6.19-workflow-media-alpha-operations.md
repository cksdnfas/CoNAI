# 26.6.19 workflow-media-alpha-operations

이 문서는 2026-06-19 KST에 승인된 `26.6.19 workflow-media-alpha-operations` 묶음 로드맵을 로컬에서 검토 가능한 상태로 정리합니다. 실제 push, deploy, tag, live demo update, server restart, protected service `3999` 조작은 포함하지 않습니다.

## 목표

- 모듈 워크플로 운영자가 저장, 복제, 재실행, 오류 복구, 결과 비교 흐름을 한 작업선으로 확인할 수 있다.
- 미디어 라이브러리 검수자가 검색, 그룹, 유사도, tag 품질, rating, recycle/recover, batch review를 non-destructive 흐름으로 처리할 수 있다.
- 제한 알파 데모 준비자가 권한, MCP client 계약, 실패 복구, smoke plan, rollback 기준을 승인 전 검토 패킷으로 확인할 수 있다.

## 작업선

### 1. Workflow operations UX

- Saved workflow browser에서 search, duplicate, reuse, run input readiness를 명확히 표시합니다.
- Execution detail은 selected path, inactive branch, skipped node, blocked downstream, failed downstream 상태를 rerun 결정과 연결합니다.
- Run comparison은 input snapshot, final result, intermediate artifact, warning context를 같은 기준으로 비교합니다.
- Rerun/retry는 backend contract가 허용하는 queue/job 경계 안에서만 노출합니다.

### 2. Media review workspace

- Search/group/similarity/tag quality/rating/review status를 같은 검수 화면에서 함께 볼 수 있게 정렬합니다.
- Batch action은 tag, rating, group, review status 같은 reversible metadata correction부터 다룹니다.
- Recycle/recover는 destructive deletion과 분리하고, deletion/retention policy는 별도 승인 경계로 남깁니다.
- Workflow result와 generation history result에서 media review workspace로 이동할 수 있는 참조 기준을 둡니다.

### 3. Limited alpha and MCP operations

- Alpha handoff는 branch, commit range, package version source, local verification, smoke plan, rollback 기준을 묶은 승인 전 패킷으로 유지합니다.
- MCP operations는 dry-run evidence, side-effect class, approvalRequired, client contract를 먼저 노출합니다.
- Live MCP call, deploy, restart, protected service `3999` 조작은 사용자 승인 전 실행하지 않습니다.
- Auth/security policy, public API expansion, credential/external billing/cloud side effect는 이 묶음 밖으로 둡니다.

## 로컬 검증

문서 묶음 기준 확인:

```bash
npm run docs:build
git diff --check
```

코드 변경이 생기는 commit-unit부터 추가 확인:

```bash
npm run verify:release-readiness
python -m graphify update .
```

`python -m graphify update .`는 코드 변경 뒤에만 자동 갱신 기준으로 사용합니다. 문서와 release note만 바뀐 경우에는 docs build와 diff whitespace 확인을 우선합니다.

## 승인 경계

- Git push, release tag, public announcement
- live demo host pull, deploy, service restart
- protected service `3999` 조작
- DB schema, retention policy, destructive cleanup
- auth/security policy change or public API expansion
- package/app version bump
- external credential, billing, cloud sync, public exposure 작업
