## Live Smoke 계획

승인 전에는 live target에 접속하거나 demo host를 변경하지 않습니다. 승인 후 smoke는 다음 순서로만 진행합니다.

1. 승인된 branch와 commit range가 이 문서의 스냅샷과 일치하는지 확인합니다.
2. demo host에서 approved alpha branch를 pull/update합니다.
3. configured demo service만 restart합니다.
4. live target `2999`에서 frontend load, backend health/API reachability, key media browsing, workflow readiness, MCP opt-in boundary를 확인합니다.
5. protected service `3999`가 변경되지 않았고 smoke 대상에서 제외됐는지 확인합니다.
6. 실패 시 추가 변경을 멈추고 rollback section 기준으로 복구 선택지를 보고합니다.

## 롤백 경계
