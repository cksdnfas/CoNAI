# 26.6.9 demo operation readiness checklist

이 문서는 `26.6.9 security-release-observability-operations` 로드맵의 M2-CU2 준비 기록입니다. 범위는 승인 후 사용할 alpha push, demo host update, restart/smoke, rollback 체크리스트를 정확히 정리하는 데까지입니다. 이 작업 중 push, pull, deploy, restart, live smoke, protected service `3999` 접근, package version bump, tag, public release action, destructive cleanup은 실행하지 않았습니다.

## 실행 전 원칙

- 이 체크리스트는 approval-gated operation packet입니다.
- 실제 외부 작업은 사용자 승인 전에는 실행하지 않습니다.
- 각 단계는 이전 단계의 결과와 승인 범위가 맞을 때만 진행합니다.
- protected service `3999`는 smoke 대상도 restart 대상도 아닙니다.
- live smoke 대상은 승인된 demo service의 live target `2999`로 제한합니다.

## 로컬 사전 확인

승인 요청 전에 로컬에서만 실행 가능한 확인입니다.

```bash
git status --short --branch
git log --oneline origin/alphatest..HEAD
npm run verify:release-readiness
npm run build
```

통과 기준:

- branch는 `alphatest`이고 upstream은 `origin/alphatest`입니다.
- working tree에는 승인 범위 밖의 dirty work가 없습니다.
- commit range는 승인 받을 handoff packet의 range와 일치합니다.
- `npm run verify:release-readiness`와 `npm run build`가 통과합니다.
- package version source는 별도 승인 전까지 `26.6.3` baseline을 유지합니다.

중단 기준:

- branch, upstream, commit range가 handoff와 다릅니다.
- dirty work가 승인 범위와 겹칩니다.
- verification이 실패하고 원인이 승인 범위 밖 변경을 요구합니다.

## 승인 후 작업 순서

| 단계 | 승인 | 명령 또는 작업 | 대상 | 확인 | 중단 조건 |
| --- | --- | --- | --- | --- | --- |
| alpha push | 사용자 승인 필요 | `git push origin alphatest` | `origin/alphatest` | remote `alphatest` HEAD가 승인된 local commit과 일치 | push target 또는 commit range가 다름, force push 필요 |
| demo host update | 별도 승인 필요 | `git fetch origin && git checkout alphatest && git pull --ff-only origin alphatest` | approved demo host only | demo host worktree가 승인된 commit에 도달 | fast-forward 불가, host branch 불일치 |
| demo service restart | 재시작 승인 필요 | restart configured demo service only | live target `2999` demo service | configured service만 재시작되고 `3999`는 untouched | 다른 서비스나 protected service `3999` 조작 필요 |
| live smoke | smoke 승인 필요 | frontend load, backend health/API, media browsing, workflow readiness, MCP opt-in boundary 확인 | live target `2999` only | 모든 smoke assertion 통과 | user data mutation, destructive cleanup, auth/security/public API 변경 필요 |
| rollback handoff | 실패 시 별도 승인 필요 | prepare rollback to previous approved commit and restart configured demo service only | approved demo branch and configured demo service | previous commit, service target, no-touch `3999` evidence 기록 | force push, destructive cleanup, protected service `3999` 접근 필요 |

## Smoke assertion detail

승인 뒤 live smoke를 실행할 때 확인할 항목입니다. 이 문서는 smoke를 실행하지 않고 assertion만 고정합니다.

- Frontend load: live target `2999`에서 앱 shell이 로드됩니다.
- Backend health/API reachability: live target에 연결된 API가 기본 응답을 반환합니다.
- Key media browsing: 기존 미디어 목록 또는 상세 흐름이 읽기 전용으로 열립니다.
- Workflow readiness: workflow/runtime 화면이 준비 상태를 보여주며 추가 실행을 강제하지 않습니다.
- MCP opt-in boundary: MCP/automation 노출은 opt-in 경계 안에 머뭅니다.
- Protected service: service `3999`는 접근, 종료, 재시작, 설정 변경 대상이 아닙니다.

## 사용자 결정

다음 항목은 agent-owned next work가 아니라 사용자 승인 결정입니다.

- `alphatest` push 여부
- demo host pull/update 여부
- configured demo service restart 여부
- live target `2999` smoke 실행 여부
- 실패 시 rollback 실행 여부
- package version bump, git tag, public release announcement 여부
- DB/data retention, destructive cleanup, auth/security redesign, public API exposure 변경 여부

## 현재 구현 반영

- Settings > Release readiness 화면에 approval-gated operation checklist가 추가되었습니다.
- 저장되는 release readiness handoff export가 `Approval-Gated Operation Checklist` 섹션을 포함합니다.
- 이 화면과 export는 준비 기록만 만들며 backend action endpoint, push, deploy, restart, smoke를 실행하지 않습니다.
