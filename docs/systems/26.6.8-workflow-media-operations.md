# 26.6.8 workflow-media-operations

이 문서는 2026-06-08 KST에 승인된 `26.6.8 workflow-media-operations` 로드맵을 로컬에서 검토 가능한 상태로 묶는 체크리스트입니다. 실제 push, deploy, tag, live demo update는 포함하지 않습니다.

## 목표

- 저장된 workflow를 찾고, 복제하고, 재사용하는 운영 경로를 설명할 수 있다.
- ComfyUI 서버 enabled/disabled 상태와 explicit Save 기반 settings persistence를 설명할 수 있다.
- failed/cancelled/completed run의 recovery, rerun, compare 흐름을 설명할 수 있다.
- generated media를 search, group, similarity, tag quality, rating, recycle/recover guardrail 관점에서 검토할 수 있다.

## 로컬 검증

npm run verify:release-readiness
```

```

## 알파 리뷰 확인 항목

- Workflow operations: saved workflow search/duplicate/reuse, ComfyUI server activation state, run-readiness guidance
- Settings operations: sections that persist settings use Save-button persistence rather than server-side save on field change
- Run recovery: queue/history states, retry/rerun affordance, compare-ready run inputs/results/warnings
- Media review: search/group/similarity/tag-quality/rating signals and non-destructive batch review actions
- Cleanup guardrails: recycle/recover is reversible by default; destructive deletion remains outside this roadmap

## 승인 필요 항목

다음은 이 readiness package의 범위를 넘습니다.

- Git push, release tag, public announcement
- live demo host pull, deploy, service restart
- protected service `3999` 조작
- auth/security policy redesign
- DB schema, retention policy, destructive cleanup
- external credential, billing, cloud sync, or public API expansion
